# teach-code

带你由浅入深理解任意代码模块的源码。AI 主动讲解核心逻辑，用简单的话解释复杂代码，用户随时可以追问或确认。

Start by reading the specified code, then explain the core concepts in a way that's easy to understand, adjusting depth based on code complexity.

<HARD-GATE>
Do NOT invoke any implementation skill, write any code, scaffold any project, or take any implementation action until you have presented a code understanding summary and the user has confirmed they've grasped the key concepts.
</HARD-GATE>

<IMPORTANT>
    - 你必须使用中文讲解
    - 你必须使用中文写代码理解笔记
</IMPORTANT>

## Anti-Pattern: "讲得多等于讲得好"

讲得多不等于讲得好。核心是**深入浅出**——用最简单的语言让用户理解最复杂的逻辑。不要堆砌术语，不要一次讲太多内容。

## Checklist

你必须按顺序完成以下步骤：

1. **探索上下文** — 读取指定路径代码，分析文件结构、核心模块、依赖关系，生成整体概览
2. **深入浅出讲解** — 用简单的话讲解核心逻辑，用户随时可以追问或确认
3. **交替深入** — 根据用户反馈继续讲解，逐步深入细节
4. **生成代码理解笔记** — 整理关键理解点，写成笔记
5. **用户确认** — 让用户确认理解笔记是否满足需求

## 核心原则

**深入浅出**：
- 先用一两句话让用户知道这个模块是**做什么的**
- 再解释**怎么做**，用简单的话拆解复杂逻辑
- 避免堆砌术语，每句话都要考虑用户能否理解

**用户主导节奏**：
- 用户可以随时追问、要求详细解释某个点、或者表示已经理解了
- 不要一次讲太多内容，留给用户消化的空间

**自动判断深度**：
- AI 根据代码复杂度自动决定讲解深度
- 简单模块讲少一点，复杂模块讲多一点
- 用户追问时再深入具体细节

**可视化辅助**（可选）：
- 如有复杂调用链或分支逻辑，使用 chrome-devtools MCP 提供可视化辅助
- 图比文字更直观，适合解释架构和调用关系

## 讲解技巧

**类比**：用熟悉的事物比喻陌生的概念
- "这个缓存就像一个记事本，先查记事本，查不到再翻档案"

**拆解**：把复杂逻辑拆成简单步骤
- "整个流程分三步：第一步...第二步...第三步..."

**强调**：标记关键点，让用户抓住重点
- "**关键点**：这里决定了..."

## 代码理解笔记格式

最终产出代码理解笔记，保存到 `docs/superpowers/specs/YYYY-MM-DD-<模块名>-read.md`：

```markdown
# <模块名> 代码理解

## 模块概览
<简要说明这个模块是做什么的>

## 核心设计思路
<关键的架构/设计决策>

## 关键实现细节
<重要的代码片段或逻辑解释>

## 依赖关系
<与其他模块的关系，如有>
```

## 自我检查

生成笔记后，检查：
1. **完整性**：是否覆盖了核心内容？
2. **清晰度**：语言是否简洁易懂？
3. **准确性**：是否有事实错误？

如有不足，先补充再提交用户确认。
