## 核心要点

1. 使用中文
2. 简短
3. 自动化执行，减少人工干预

## 工作流程

### 1. 分析变更
- 执行 `git status` 查看当前变更
- 分析变更类型（新功能、修复、重构、文档等）

### 2. 智能分支命名
基于变更类型自动创建分支名：
- **feature/**: 新功能开发
- **fix/**: Bug 修复
- **refactor/**: 代码重构
- **docs/**: 文档更新
- **style/**: 代码格式调整
- **test/**: 测试相关
- **chore/**: 构建/工具/配置

命名规则：`{类型}/{简短描述}-{日期}`，例如：`feature/add-login-0305`

### 3. 分支管理
- 从当前分支创建新分支
- 自动创建新分支并切换

### 4. 提交代码
- `git add .`
- 生成符合 commitizen 格式的 commit message
- `git commit`

### 5. 自动推送
- 自动设置 upstream：`git push -u origin {分支名}`
- 推送完成后显示分支链接
- 使用 chrome devtools 自动打开浏览器这个分支链接

## 特殊情况处理

- **分支已存在**：自动切换到已有分支
- **冲突处理**：如遇到冲突，提示用户解决
- **空提交**：如无变更，提示用户
- **推送失败**：自动重试一次，仍失败则提示手动处理
