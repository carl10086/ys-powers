# teach-code 命令设计

## 背景

用户希望创建一个交互式代码理解命令，能够通过 **AI 主动讲解 + 用户随时追问** 的方式，带你由浅入深理解任意代码模块的源码。核心是**深入浅出**——用最简单的语言让用户理解最复杂的逻辑。

## 目标

- 用户随意指定文件/目录/模块作为起点
- AI 根据代码复杂度自动决定深入程度
- **AI 主动讲解**核心逻辑，用户随时可以追问或确认
- 复杂场景支持可视化辅助（chrome-devtools MCP）
- 最终产出代码理解笔记

## 设计决策

### 核心定位

**从"AI 审问"变为"AI 主动教"**：

| 旧思路（审问式） | 新思路（教学式） |
|----------------|----------------|
| AI 问问题 | AI 主动讲解 |
| 用户回答 | 用户随时追问 |
| 生成笔记 | 过程中就完成理解 |

### 核心流程

```
用户输入：/teach-code <文件/目录/模块路径>

Step 1: 探索上下文
  → AI 读取指定路径代码，分析文件结构、核心模块、依赖关系
  → 生成整体概览（中文输出）

Step 2: 深入浅出讲解 + 用户追问
  → AI 用简单的话把核心逻辑讲清楚
  → 用户可以随时追问或要求详细解释某个点
  → 交替进行，逐步深入
  → AI 自动判断深度

Step 3: 总结
  → 生成代码理解笔记
  → 保存到 docs/superpowers/specs/YYYY-MM-DD-<模块名>-read.md
```

### 深入浅出的技巧

- **类比**：用熟悉的事物比喻陌生的概念
- **拆解**：把复杂逻辑拆成简单步骤
- **强调**：标记关键点，让用户抓住重点

### 文件结构

```
ys-powers/
├── commands/
│   └── teach-code.md         # command 文件
└── docs/superpowers/specs/
    └── YYYY-MM-DD-teach-code-design.md  # 本设计文档
```

## 验证标准

- `commands/teach-code.md` 存在
- 流程清晰：AI 主动讲解 → 用户追问 → 总结笔记
- 强调深入浅出原则
