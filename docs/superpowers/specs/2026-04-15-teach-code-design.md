# teach-code 命令设计

## 背景

用户希望创建一个交互式代码理解命令，带你由浅入深理解任意代码模块的源码。

核心定位：**不是帮你复习看过的代码，而是带你理解你没看过的代码**。

## 核心原则

### 讲什么

- **功能**：这个代码是做什么的
- **设计**：它是怎么组织的，为什么这样组织
- **实现**：用伪代码/文字描述讲逻辑，不讲语法细节

### 怎么讲

- 从宏观到细节，拆成一个个点
- 用伪代码代替逐行语法讲解
- 每个模块用固定格式：是什么 → 怎么组织 → 怎么做

### 问什么

- 追问设计动机："为什么要这样设计？"
- 追问权衡："这样设计有什么缺点？"
- 追问替代："有什么其他方案？"

## 讲解层级

| 层级 | 内容 | 重点 |
|------|------|------|
| 模块是做什么的 | 核心功能、系统角色 | 功能 |
| 怎么组织的 | 组成部分、关系、数据流动 | 设计 |
| 具体怎么实现的 | 伪代码/文字讲逻辑 | 实现 |

## 讲解格式

每个概念/模块，用这个格式讲：

```
📍 是什么：它的功能
🔧 怎么组织：设计思路
⚙️ 怎么做：用伪代码讲逻辑
```

## 问题质量

好问题引发思考：
- "为什么要这样设计？"
- "这样设计有什么缺点？"
- "有什么替代方案？"
- "如果不用这个设计，会怎样？"

Never 问：
- "你知道这个 TypeScript 语法是什么意思吗"
- "你理解了吗"

## 验证标准

- `commands/teach-code.md` 存在
- 核心节奏：讲一段问一段
- 讲解内容聚焦：功能→设计→实现（不用讲语法）
- 问题质量：追问动机、权衡、替代
